## Summary

Build a frontend-only appointment booking page at the `/book` route for Dr. Shenouda Dental Clinic. The page displays a schedule of available time slots (every Saturday and Wednesday at 4:00 PM local time, for the next 8 weeks), allows patients to select a slot and submit their name, phone, and email to book it. Bookings are persisted in browser localStorage with a capacity of 1 per slot. The page supports bilingual Arabic/English via the existing i18n system, uses the same design tokens and Tailwind CSS styling as the landing page, and is architected with a data source abstraction layer to allow future integration with external feeds (Google Calendar/Excel). No backend or server-side processing is required.

## Technical Context

**Language/Version**: JavaScript (ES2022) + React 18.3, JSX via Vite 5

**Primary Dependencies**: React 18, react-router-dom 6 (new), Tailwind CSS 3.4, Framer Motion 11, Google Fonts (Cormorant Garamond, DM Sans, Cairo), Material Symbols Outlined

**Storage**: Browser localStorage — bookings stored as JSON array under key `dr-shenouda-bookings`. No backend database.

**Testing**: Browser-based manual validation against acceptance scenarios in spec.md. DevTools localStorage inspection for data persistence verification.

**Target Platform**: Modern evergreen browsers (Chrome, Firefox, Safari, Edge — latest 2 versions). Mobile-first responsive (320px–1920px).

**Project Type**: web-application (multi-page SPA — landing page + booking page)

**Performance Goals**: Booking page interactive within 2 seconds; slot selection and form submission feel instant (localStorage is synchronous); schedule renders all 16 slots without layout shift.

**Constraints**: No backend/API/database; no cancellation in MVP; localStorage only (5MB limit, ~10,000 bookings before quota); dates in DD/MM/YYYY format; one booking per slot; existing landing page BookingSection remains unchanged.

**Scale/Scope**: Single booking page with 3 components (ScheduleView, BookingForm, BookingSummary), 16 time slots max displayed, bilingual (ar + en), 1 data source for MVP.

New booking-specific components live under `src/components/booking/` to keep them separate from landing page components. A new `src/pages/` directory holds page-level route components. The existing landing page structure is completely untouched except for minor modifications to `App.jsx` (routing), `main.jsx` (BrowserRouter wrapper), i18n files (new keys), and validation utils (extended phone regex).
